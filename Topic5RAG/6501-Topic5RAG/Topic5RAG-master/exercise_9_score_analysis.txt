======================================================================
RECOMMENDATIONS
======================================================================

1. USE SCORE GAPS AS CONFIDENCE INDICATORS:
   - Gap > 0.1: High confidence, use top 3-5 chunks
   - Gap 0.02-0.1: Medium confidence, use top 5-7 chunks
   - Gap < 0.02: Low confidence, use top 10 or expand query

2. IMPLEMENT DYNAMIC THRESHOLDING:
   - Set threshold relative to top score (e.g., top_score - 0.2)
   - Or use percentage of top score (e.g., 70% of top)
   - Better than fixed threshold

3. MONITOR SCORE DISTRIBUTIONS:
   - Track average scores per query type
   - Flag queries with unusually low scores
   - Use for system quality monitoring

4. COMBINE SCORES WITH OTHER SIGNALS:
   - Don't rely solely on similarity scores
   - Consider query type, document structure
   - Use reranking for final selection

5. DEFAULT THRESHOLD GUIDELINES:
   - Conservative: 0.6 (high precision, may miss relevant)
   - Balanced: 0.5 (recommended default)
   - Permissive: 0.4 (high recall, more noise)
