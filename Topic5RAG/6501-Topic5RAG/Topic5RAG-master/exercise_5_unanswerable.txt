======================================================================
ANALYSIS
======================================================================

Key Findings on Unanswerable Questions:

1. DEFAULT PROMPT BEHAVIOR:
   - Model sometimes hallucinates plausible-sounding answers
   - May combine retrieved context with general knowledge incorrectly
   - Doesn't always clearly state when information is missing
   - Uses training data to fill gaps when context is insufficient

2. STRICT PROMPT IMPROVEMENT:
   - Explicit instruction to say "I cannot answer" helps significantly
   - Model more likely to admit when context doesn't contain the answer
   - Reduces hallucination on out-of-scope questions
   - Forces grounding in retrieved context only

3. CATEGORY ANALYSIS:

   OFF-TOPIC QUESTIONS:
   - Usually handled well (obviously not in corpus)
   - Both default and strict prompts correctly identify scope mismatch
   - Low hallucination risk

   RELATED BUT MISSING:
   - High hallucination risk
   - Model may use training data to "fill in" gaps
   - Strict prompt helps but doesn't eliminate all issues
   - Model may conflate similar information

   FALSE PREMISES:
   - Model often catches anachronisms (electronic ignition in 1919)
   - But may still try to answer rather than rejecting premise
   - Strict prompt improves rejection rate

4. RETRIEVAL IMPACT:
   - Retrieved context sometimes helps (model sees no relevant content)
   - Sometimes hurts (model tries to force connection)
   - Strict prompting with context check reduces false connections

5. HALLUCINATION INDICATORS:
   - Specific values without context citation
   - Anachronistic technology references
   - Confident statements about obviously missing information
   - Use of "According to the manual" when manual doesn't say that

======================================================================
RECOMMENDATIONS
======================================================================

1. USE STRICT PROMPTING:
   - Explicitly instruct model to admit ignorance
   - Reduces hallucination significantly
   - Essential for high-accuracy applications

2. ADD CONFIDENCE SCORING:
   - Use retrieval score thresholds
   - Filter chunks with score < 0.5
   - Flag low-confidence retrievals

3. IMPLEMENT QUERY CLASSIFICATION:
   - Separate classifier to detect off-topic questions
   - Pre-filter before retrieval
   - Better than relying on model to self-report

4. MONITOR FOR HALLUCINATION:
   - Check citations against retrieved chunks
   - Verify specific values appear in context
   - Flag answers that go beyond retrieved content

5. PROVIDE FALLBACK:
   - When confidence is low, offer to search other sources
   - Don't force an answer from insufficient context
   - Better to say "I don't know" than to hallucinate

BEST PRACTICE:
Always use strict prompting for production RAG systems where accuracy
is critical. The slight reduction in helpfulness is worth the
significant improvement in reliability.
